Role Permissions Explained

All role permissions are listed below. Permissions are added to roles (not users). Roles are assigned to users/groups.

Note: In SaaS with b2c authentication, groups are not used.

Not all permissions are available for all role types. For example, the permission Manage Security is only available for the role types eDiscovery (Legal Hold & Legal Review) and Legal Review, not for the role types (Current) Matter.

First, you select a role type; you can select an existing role or create a new one. Then, add/remove permissions (listed below) to/from that role.

Some permissions are related/connected to other permissions. You will get a warning when you select a related permission. It is advised to select the suggested permissions in order for the selected permission to function properly.

Review

  • Access all documents
    This role permission can be defined for a Legal Review Role. Access is granted for all documents in all matters.
  • Access all fields
    This role permission can be defined for a Legal Review Role. The user is allowed to view all field properties.
  • Access document content
    When created in a Legal Review Role, the user is allowed to open all content available in Matters in ZyLAB Legal Review. For Current Matter Roles, only the content of one Matter (the current or specified matter) can be opened.
  • Access documents
    • Not granted
    • Granted for all documents
    • Granted for specific documents
  • Access fields
    • Not granted
    • Granted for all documents
    • Granted for specific documents
  • Access matter
    When created in an eDiscovery or Legal Review Role, the user is allowed to open all Matters in ZyLAB Legal Review. For Current Matter Roles, only one Matter (the current or specified matter) can be opened.
  • Add/edit remarks
    The user is allowed to add and edit remarks to reviewed documents.
  • Annotation - annotate documents
    The user is allowed to annotate documents.
  • Annotation - edit annotations created by other people
    • Not granted
      No users are allowed to edit annotations created by other people.
    • Granted for annotations created by anybody
      All users are allowed to edit annotations created by other people.
    • Granted for annotations created by specific people
      Specified users are allowed to edit annotations created by other people.
  • Annotation - see annotations created by other people
    • Not granted
      No users are allowed to see annotations created by other people.
    • Granted for annotations created by anybody
      All users are allowed to see annotations created by other people.
    • Granted for annotations created by specific people
      Specified users are allowed to see annotations created by other people.
  • Bulk field update
    The user is allowed to use the Batch Update function in Batch Search. If the user does not have this permission, the update functionality and columns (Updated Documents, Value) are hidden.
  • Create Batch Downloads
    Allows users to create, download, view and work on their own batch downloads (selection of documents). 
  • Manage Batch Downloads from other users 
    Allows users (administrators) to view/work on batch downloads created by other users. With this role permission, you can work on all batch downloads created in a matter.
  • Manage Cloud Upload
    Allows users to manage cloud uploads (Microsoft Exchange, Google G Suite).
  • Create matter
    The user is allowed to create matters.
  • Create Sample Set
    The user is allowed to create a sample set.
  • Manage All Sample Sets
    The user is allowed to manage all sample sets.
  • Deduplication
    The user is allowed to perform deduplication.
  • Delete documents
    The user is allowed to delete documents.
  • Download documents
    The user is allowed to download files in the native or PDF format.
  • Redact documents
    The user is allowed to protect sensitive, privileged and/or confidential information by hiding selected content.
  • Review batches
    The user is allowed to create review batches (workloads).
  • Search
    The user is allowed to search the document set, including metadata.
  • Tagging - apply tags
    The user is allowed to apply any tags.
    • Not granted
    • Granted for all tags
    • Granted for specific tags
  • Tagging - bulk tag
    The user is allowed to apply bulk tags.
  • Tagging - see tags
    The user is allowed to see all tags.
    • Not granted
    • Granted for all tags
    • Granted for specific tags
  • Upload
    The user is allowed to upload files.
  • View audit log
    The user is allowed to analyze the actions of users.

Assisted Review

  • Assisted Review
    The user is allowed to use Assisted Review.

Configuration

  • Auto-redact documents
    The user is allowed to auto-redact the document he is currently reviewing.
  • Batch assignment
    The user is allowed to assign batches to legal reviewers.
  • Configure document auto-classification
    The user is allowed to predefine a list of document classification rules. If an Auto-Classification rule (query) finds a match, field values or tags of the document will be adjusted.
  • Configure keyword highlighting
    The user is allowed to configure groups of keywords to be highlighted.
  • Configure review status query
    The user is allowed to define a review status query. When a document matches the review status query, the document will be automatically marked as reviewed.
  • Delete matter
    When created in a Legal Review Role, the user is allowed to delete all available Matters in ZyLAB Legal Review. For Current Matter Roles, only one Matter (the Current Matter or specified Matter) can be deleted.
  • Manage facets
    The user is allowed to manage facets.
  • Manage security
    Only available for eDiscovery and Legal Review Roles, not for Current Matter Roles. The user is allowed to control security configurations for the selected matter. For Matter Admin Roles, the permission is enabled by default.
  • Tagging - edit configuration
    The user is allowed to configure the tags.

Production

  • Produce documents
    The user is allowed to deliver relevant, non-privileged data sets to opposing counsel, partner firms, outside counsel and/or the requesting government agency.
  • Access produced document content
    The user is allowed to view if and how documents are produced.

Reports

  • Access all internal reports
  • Access reports
    The user is allowed to access/view reports.
    • Not granted
    • Granted for all reports
    • Granted for specific reports
  • Access SSRS reports
    The user is allowed to access/view SSRS (SQL Server Reporting Service) reports.

Insights

  • Configure Insights
    The user is allowed to configure Insights.
