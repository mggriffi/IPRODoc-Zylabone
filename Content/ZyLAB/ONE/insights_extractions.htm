Extractions

Create and run extraction jobs to automate the extraction of entities, as defined by the extraction rules applied.

 

  1. Open Home > Insights > Extractions.

  2. Create a New Extraction.

  3. Define the Name; enter a distinguishable, easily recognizable name the name field.

  4. By default, Schedule for incoming documents is enabled.

    • When enabled, the Extraction job will run on existing and future documents.

    • When disabled, the Extraction job will run only on existing documents.

    Tip: To run the job only on future documents, use the Query Builder to create a new Query; set the document date 'from' to follow the current date, and add the desired filter(s).

  5. (Optional) You can enable/disable the Schedule for incoming documents option for existing or previously created Extraction jobs.

    • If switching from enabled to disabled, the job will no longer run on future documents.

    • If switching from disabled to enabled, the job will run only on future documents; only future documents will be processed. In addition, all documents will be rescheduled for processing.

      Note I: When Schedule for incoming documents is disabled, any documents that are added later, will not be processed. When Schedule for incoming documents is enabled, only documents that have yet to be added, will be processed.

      Documents added whilst the Schedule for incoming documents option was disabled, will not be processed when Schedule for incoming documents is enabled.

      To ensure that there are no discrepancies in the document processing, you can create and run a new Extraction on all existing and all incoming documents – create and run an Extraction with the Schedule for incoming documents option enabled.

      Note II: If you have completed or are running an Extraction prior to updating to VX ZyLAB ONE, consider that there can be discrepancies in the document processing.

      When migrating to VX ZyLAB ONE, the Schedule for incoming documents option will be disabled for all migrated Extractions. If you then decide to: I) upload a new set of documents and II) enable the Schedule for incoming documents option, there will be a discrepancy in document processing; the documents added whilst the Schedule for incoming documents option was disabled, have not been processed – and will not be processed when the Schedule for incoming documents option is enabled.

  6. Select the Extraction Rules to be applied.
  7. (Optional) Review the Documents in scope; The Scope defines the estimated character count and the exact number of documents that will be analyzed.

  8. Note: Performing extractions on a whole matter is not always needed and can be very time consuming.
    If needed, adjust the Scope by defining a Query.

  9. When finished, click Apply. The newly created Extraction job will start running.

    Note: If an entity is being deleted and the status beside it shows 'Deleting', then you cannot schedule another job for the same entity until the deletion is completed.

  10. Review the extractions list to view a list of all extraction jobs and their respective extraction rules. Review when each extraction job was last run. You can also Filter by:

    Job Status:

    Select one or more:

    • In Progress

    • Cancelling

    • Cancelled

    • Completed

    Name:

    Enter the name of the job you wish to view.
