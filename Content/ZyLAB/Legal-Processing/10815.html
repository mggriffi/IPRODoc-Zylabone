Failed Actions Report

At opening the Failed Actions Report, the current failed actions of the selected matter and the reasons of failure are listed. With the Refresh button, the latest situation will be displayed. Failed actions that succeed after restart, no longer appear in the report.

The following information is provided:

  • ID of the failed action
  • Machine name
  • Performer name
  • Performer configuration name
  • SourcePath
  • Last Original file name (first part of extraction path) (original Name within the last container)
  • Current file location (on the shared location)
  • Custodian
  • Source
  • JobID
  • Mimetype
  • DocumentID
  • Filesize
  • Retries (number of restarts, if restarted)
  • TimeStarted
  • TimeFinished
  • ExcecutionTime
  • Error description
  • Total Count
  • Filtered Count

If there are several errors for one action, only the last one will be displayed with information about the number of restarts of this action.
