SharePoint Collector

The SharePoint Collector collects objects from SharePoint Online and SharePoint On Premise (Intranet-based solutions through Client Object Models and SharePoint web services). ZyLAB offers support for SharePoint 2010, 2013, and SharePoint Online.

In the ZyLAB.EDiscovery.Collectors.SharePoint.dll.xml config file all property names that can be collected are specified. Here you can define the metadata (field properties) that should be collected. For each property you can set the Extract value to True (it will be collected) or False (it will not be collected). As the collector is designed to collect from different versions of Sharepoint (for which Sharepoint sometimes uses different property names), some properties are added more than once with a different name (for example, Author and _Author).

XML Configuration

We collect the following types of data (column types):

  • Single line of text
  • Multiple lines of text
  • Choice
  • Number
  • Data and Time
  • Lookup
  • Person or Group
  • Hyperlink or Picture
  • Managed Metadata

Client Configuration

  • Credential Type
    Choose from ClaimsBased (used to connect to SharePoint Online), Ntlm (windows authentication), Basic or Digest (used to connect to SharePoint on premise). Depending on how IIS is configured, you choose Ntlm, Basic or Digest (Go to your Sharepoint site folder in IIS and check the Authentication that is being enabled).

  • Username
    Define the username used to access SharePoint.
    For example, for SharePoint Online: somebody@company.onmicrosoft.com
    For example, for SharePoint on premise: Somebody

    This user should have Read Rights in SharePoint (Visitor Group).

  • Password
    Define the password related to the user defined for SharePoint Online or SharePoint on premise.
  • Domain
    Only to be defined when using Ntlm, Basic or Digest.
    For example: zylab

  • Proxy Realm
    Optional. Only used for Social Tags & Notes collection via SharePoint Web Services.
  • Location
    Define the URL of the SharePoint site.
    For example, for SharePoint Online: https://company.sharepoint.com/sites/somesite/
    For example, for SharePoint on premise: http://intranet.company.com
  • Libraries (optional)
    By default, files from all libraries will be collected. If you do not want that, you can add specific libraries.
    For example 'Shared Documents', 'Temporary Folder'.
  • Include Older Versions
    Select if you want to collect all versions of a document. Do not select if you want to collect only the latest version of a document.
  • Date Range (optional)
    Optionally define the date range of the objects you want to collect.

    The date range is not dynamically adjusted. For example, when you choose to collect everything until today (inclusive) and want to rerun the same collection action the next day, it will collect everything until yesterday (inclusive).

  • File Types to Collect
    Select the appropriate File Groups. The selected File Groups are used as a filter to determine which documents need to be collected.
    To create new File Groups, click Add New.
    For more information on how to define File Groups, see Define File Groups.

Social Tags & Notes



The SharePoint Collector also collects Social Tags and Social Notes from documents in SharePoint. They are collected via calls to SharePoint Web Services. By default this information is not displayed in Legal Review. If you want this information to appear in Legal Review, set the Extract value for the property names 'SocialTags' and 'SocialNotes' to True in the xml config file, and add the fields to the Legal Review template.

Configuration SharePoint Data Retrieval through Client Object Models

SharePoint Online is currently restricted to communication through SharePoint Foundation 2010 Client Object Models provided by Microsoft (current version used 15.0)

In the SharePoint Client Object Models, the ClientContext object serves as a gateway for accessing the SharePoint site. The object is used for everything from accessing data to performing tasks.

  • In order for client context to be able to communicate with server, the authenticated cookies need to be attached.
  • Client context is used to query for SharePoint libraries.
  • Client context is used to query for items in a specific library.
