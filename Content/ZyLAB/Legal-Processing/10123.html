Email Collector (Default & Enterprise)

The Email Collector collects email from email providers that support IMAP, POP3 or ActiveSync.

First, select the Authentication Type (Social or Enterprise). Social is used to collect items from a user with a known password. Enterprise is used to collect items from a user or group of users in the same domain, without knowing all the passwords. Instead, you will use the Customer Key and Customer Key supplied by the Administrator.

Second, select the provider (IMAP, POP or ActiveSync).

IMAP

IMAP is supported by Gmail, Aol and Yahoo. Only IMAP is available as provider for the authentication type Enterprise (the other being the Social type or default configuration).

POP

POP3 is supported by Hotmail and Yahoo Plus!. Because of the limitations in the POP3 protocol, full text search is not possible on the body of the email and only messages from the Inbox can be collected.

  • Settings

    Email
    Define the full email address (for example, for Hotmail: username@hotmail.com, for Yahoo Plus!: username@).

    Password
    Define the password related to the Hotmail or Yahoo Plus! email address.

    Host
    pop3.live.com or plus.pop.mail.yahoo.com

    Enable SSL
    Yes

    Port
    995

ActiveSync

ActiveSync is supported by Gmail, Hotmail and Exchange. The ZyLAB Exchange ActiveSync settings are not yet supported. Full text search is not yet implemented.

  • Settings

    Email
    Define the full email address (for example, for Gmail: username@gmail.com, for Hotmail: username@hotmail.com).

    Domain\Username
    For example, for Gmail: google\username@gmail.com, for Hotmail: username@hotmail.com

    Password
    Define the password related to the Gmail or Hotmail email address.

    Host
    m.google.com or m.hotmail.com

    Internet email messages consist of two major sections:
    - Header (structured into fields such as From, To, CC, Subject, Date, and other information about the email). Each field has a name and value. RFC 5322 specifies the precise syntax (http://tools.ietf.org/html/rfc5322 ).
    - Body (the basic content, as unstructured text; sometimes containing a signature block at the end)


    Header
    The Header filter is currently only supported for IMAP. If you use this field for POP3 or ActiveSync, you will get a warning when checking the matter in.

    To define an Email Header Field filter, use the following format:

    'Header1:value1 value2|Header2:value3'

    For example:

    Date:7 Mar|MIME-Version:1.0|X-Priority:3

    From
    Collect messages sent from someone. Define a user/email address.

    To
    Collect messages sent to someone. Define a user/email address.

    Cc
    Collect messages sent to someone using the CC field. Define a user/email address.

    Bcc
    Collect messages sent to someone using the BCC field. Define a user/email address.

    Subject
    Collect messages by the words specified in the Subject.

    Body
    Collect messages by the words specified in the Body.

    Date Range
    Collect messsages by a defined date range.

    The date range is not dynamically adjusted. For example, when you choose to collect everything until today (inclusive) and want to rerun the same collection action the next day, it will collect everything until yesterday (inclusive).
