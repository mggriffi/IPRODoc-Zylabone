ZyLAB Batch Assignment and TIFF Pre-Generation Tool

The ZyLAB Batch Assignment and TIFF Pre-Generation Tool enables you to improve the review performance, keyword highlighting and automatic redactions. The Batch.exe file can be scheduled under the ZyLAB folder of the Windows Task Manager (C:\Program Files (x86)\ZyLAB\EDiscovery\LegalReview\Tools). There are 4 functions that can be executed with parameters. These are:

  • Batch assignment (according Legal Review batch assignment rules)
  • Tiff pre-generation (according to specified query parameter of this command)
  • Pre-generation of the keyword highlighting hits (according Legal Review keyword group definitions)
  • Autoredaction (according Legal Review autoredaction rules)

More than one function can be chosen in parallel, according to one matter-id parameter.

Usage

batch [assign] [pregen] [concepthits] -svc <ClientServiceUri> [-m <MatterId>] [-q <QueryText>] [-tc <ThreadsCount>] [-t <TimeLimit>] [-v]

Modes

assign

Perform batch assignment

pregen

Perform tiff pre-generation

concepthits

Perform pre-generation of the keyword highlighting hits

autoredactions

Perform pre-generation of autoredactions

Settings

-svc <ClientServiceUri>

Uri for the client service facade of Legal Review.

-m <MatterId>

Id of matter to work on. If not specified all matters will be processed.

-q <QueryText>

Query to match documents for tiff pregeneration. Default: AssignedTo <> null and Reviewed = false.

-tc <ThreadsCount>

Number of concurrent documents to be pregenerated. Default: number of cores on your machine.

-t <TimeLimit>

Time span to perform tiff pregeneration in the following format: HH:MM:SS. You can also specify it partially: HH:MM or HH. Required if pregen mode is enabled.

-v

If specified the log messages will be written to console (default is false).

-batch

Batch size (default is 10000)

-rc

LR access retry* count number (default is 30)

-rd

LR access retry* delay seconds (default is 60)

* Only used in case of access issues
