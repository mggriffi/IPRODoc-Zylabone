Field Appender

The Field Appender is a processor that allows users to append properties to the metadata fields of each file.
In the Field Appender's XML configuration file (\\Program Files\ZyLAB\EDiscovery\Processors\ZyLAB.EDiscovery.Processors.FieldAppender.dll.xml) three types of properties can be assigned:

<FixedProperty name="SomeProp" value="SomeVal" />

Here users can add a static property. For example, to add a field to all the files with the content Reviewed-No, a user can add a property like this:
<FixedProperty name="Reviewed" value="No" />

<BinaryHashProperty name="DedupSHA5BinHash" hashType="SHA512" />

Here, users can specify and add a hash type (SHA512, SHA3, SHA2, SHA1 or MD5) for files in the property. It will create new property in the metadata of each binary of each document processed. In this example, the name of property will be "DedupSHA5BinHash", SHA512 hash will be calculated from the content of the binary and this hash will be assigned as a value of this property.

This field (value) will be added by default.

<MetadataHashProperty name="SomeMetaHash" propertiesHashed="Original Folder|Original File Name|Created Timestamp|Size" hashType="MD5" />

Here, users can calculate a hash value for a few (or all) of the properties extracted for a file. Supported hash algorithms are SHA5, SHA3, SHA2, SHA1 and MD5. It will create a new property in the metadata of each binary of each document processed. In this example, the name of the property will be "SomeMetaHash", the hash value will be calculated from the contents of values of 4 other properties of current binary (Original Folder, Original File Name, Created Timestamp, Size) and this hash will be assigned as a value of this property. The hash value does not depend on the order in which the properties are specified or appear in metadata of binary.

Note 1: The fields are only added to the XML Wrapper of the document. To allow users to search these fields, they should be added to the index in ZyINDEX, prior to building the index (otherwise the field values created by the Field Appender will not be added to the index and cannot be searched).

Note 2: You can add several properties of same type, e.g. 2 Fixed Properties, 3 Binary Hash Properties and 15 Metadata Hash Properties.
