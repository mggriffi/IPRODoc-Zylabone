Define Custodians

Conditions

In a matter the scope of the responsive data is described. During the meet-and-confer process parties involved define key players/custodians. A custodian is a person empowered by statute with the care and custody of records, books and papers filed or deposited in their respective offices or at other locations. These people need to be connected to the matter.

Instructions

  1. ZyLAB Legal Processing is open.
  2. If the relevant matter is not yet opened, search and open it. The latest matter is shown first.
    You can search for a matter via the tabs on the left side of the screen:

    • All
      All created matters will be shown.
    • Active
      Active matters are all opened and being worked on.
    • Inactive
      Inactive matters are all closed. Nobody is working on them.
    • Checked-out
      Checked-out matters can be open (active) or closed (inactive), but only the person who has checked a matter out can open it.
    • Checked-out by me
      Checked-out by my matters are matters from the logged-in user. They can be open (active) or closed (inactive). Only the logged-in user can open and work on them.
  3. Select the Collection tab.
    There are two ways to add Custodians: a) Add Using Wizard (or, when creating a new matter, by selecting Add location in the Wizard) or b) Add custodians one-by-one.
  4. Click Add Using Wizard, enter a name of a custodian or part of it and click 'Resolve' to search for this name in the Active Directory. To add default sources (to define them, see Edit Global Settings), select Add default sources. Click Next.


  5. Define a source name and click 'Add'. Click Next.

  6. Define and add one or more Locations (see Define Sources and Locations). Select File Types to collect (to define them, see Define File Groups, step 5). Click Finish.
