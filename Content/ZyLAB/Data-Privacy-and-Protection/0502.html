Process with ZyLAB ONE

The process of handling personal data breaches with ZyLAB ONE is very much like the process of handling a data subject access request (DSAR). ZyLAB has an extensive instructional section regarding this process. Only four steps of this process are slightly different.

Refer to the following link for the entire process—Data Subject Access Request

The four slightly different steps are outlined and explained in these topics:

  • Search
  • Tag
  • Redact
  • Produce Documents

First, due to the request from a data subject, the DSAR process starts. The organization then has a month to comply and respond to the request. However, in the case of a personal data breach, time is of the essence. Moreover, the search does not need to be as thorough and detailed as DSAR. What is important here is to estimate the categories of personal data that were affected by the breach, the categories of data subjects, and the approximate number of data subjects and their data involved.

This process becomes much easier with ZyLAB. All you need to do is upload the data into ZyLAB ONE, search through the data, tag them, and produce a report. No redaction is necessary for these purposes.
