Decide

After the appropriate tags have been assigned to the documents, you need to decide what you want to do with the data. 

For instance, if a document set has the Purpose tag attached to it, it means that the documents have a purpose for processing and can be stored in accordance with the GDPR.

In this use case, we highlight three actions that ensure personal data is handled in accordance with the GDPR:

  • Adding purpose and storing

  • Anonymizing 

    Removing the option for the data subject to be attributed to personal data within a document. The data is irreversibly altered and it is impossible to detect if data relates to two, three or more people. The GDPR does not see it as personal data anymore.

  • Pseudonymizing

    Encrypting personal data by replacing the real personal data with a pseudonym. The main difference is that pseudonymization is a reversible process, unlike anonymization. Also, the real person is identifiable with the proper key, which makes the data still valuable for an organization. However, this also means that the GDPR still sees pseudonymized data as personal data.


The choice between anonymization or pseudonymization depends on many factors (the use case, degree of risk, the way data is processed within your company, etc.).
