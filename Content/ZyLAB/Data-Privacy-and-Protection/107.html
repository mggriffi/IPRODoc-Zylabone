Search

Uploaded data needs to be categorized. GDPR makes a distinction between personal data and special category personal data, and different measures apply to the different categories of data. The first step is deciding how you want to search the documents. There are many ways to search through data in ZyLAB ONE, the most straightforward way is using the search bar.

First, select a Matter and press Browse. Use the Search Bar to locate the (personal) data you are looking for. Enter keywords, boolean operators, proximity operators, wildcards, and more. For example, you can type a specific phone number in the Search Bar; results will include all documents with that phone number in it. For more information about all possible operators—see the ZyLAB ONE Search Language Guide.

    

  • When active, the Query Builder button is blue.
    The Query Builder assists you when creating complex queries.

  • Combine search terms with operators such as: AND,  OR or NOT.
    Other suggestions are: W/5 (Within), P/5 (Precedes), ~2 (Fuzzy), () (Parentheses),
    [0-9a-z]{3} (pattern), 2 OF {query1, query2, query3} (Quorum).

    Tip: Select Parentheses and immediately start typing (the cursor is automatically placed inside the brackets).

  • Combine full-text and Fields/Tags/Review Status queries with AND or OR. Select the button to switch from AND to OR and back.
    The AND/OR button is gray when not active (that is, not combined with another query).

      Important: Take note of the current limitation: You cannot add Tags/Review Status queries to sub queries.
