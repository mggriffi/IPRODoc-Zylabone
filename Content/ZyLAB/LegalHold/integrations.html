Integrations

The Integration page contains an overview of all the current integrations that are in the subscriptions.

The following integration types can be shown:

  • Microsoft 365 Office Email, Google Workspace Email or SendGrid Email

    Shows the users that allowed sending emails on their behalf or perform synchronizations of the Address Book. SendGrid Email is the email system of LEGAL HOLD and can be used when Microsoft 365 Office Email and Google Workspace Gmail are not available.

    These integrations can only be viewed. It gives the Administrator insight into the integrations and whether the integrations are valid.

    Select Manage your own integrations (top right corner) to enable or disable your own email preferences or to send a test message.

  • Microsoft Purview (In-place Preservation)

    Preserving data in-place means you retain the data where it is actually being used, rather than moving it to a separate archiving solution or to a separate location.

    supports preservation in-place for Microsoft Purview. This means that for custodians that are part of a hold, you can preserve (put on hold) their mailboxes in Microsoft 365.

    Note: Please view the important Notes for Administrators on the Microsoft Purview Integration.

  • Contact Synchronization (Microsoft Entra ID, Okta, and Workday)

    offers the ability to synchronize contact details in the Address Book with Microsoft Entra ID, Okta, and Workday.

    By synchronizing contacts with the company's employee directory, you will always work with an up-to-date list of potential custodians. (Contacts become custodians when added to a hold.)

Create Integration

  1. Select Create Integration.

     

  2. Select the Integration Type and the Tenant. Select Next. 

  3. Authenticate the integration. First, copy the code. Then, go to https://microsoft.com/devicelogin

  4. Paste the code and click Next.

  5. Sign in with the account that has sufficient privileges to preserve data in-place in your Microsoft 365 subscription. You should have either Compliance Administrator (formerly known as eDiscovery Manager) or eDiscovery Administrator Role permissions. 

    For more information, please refer here (only available for Administrators): 

    https://compliance.microsoft.com/compliancecenterpermissions

  6. Click Continue.

  7. Once you have authenticated, the window can be closed and you can return to Create Integration wizard.

    Click Save to finalize the integration.

  8. Once the connection is established, a test is being executed (creation and removal of an in-place hold) to check if the integration has sufficient privileges. Select the Refresh icon (to move the status from Validating to Enabled).

  9. View the validation report with the test results by clicking the Show Validation Results button.

  10. As the authentication is limited in time, the Microsoft 365 Compliance Center Integrations can expire. Therefore you have a renew option for the integration.

  11. With the integration completed, you can now create Preservation Locations to determine which locations need to be preserved in the hold.
