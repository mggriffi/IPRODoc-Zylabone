Integrate with Azure Active Directory

In this topic, you'll learn how to integrate ZyLAB ONE Legal Hold with Azure Active Directory (Azure AD). When you integrate ZyLAB ONE Legal Hold with Azure AD, you can:

  • Control in Azure AD who has access to ZyLAB ONE Legal Hold.

  • Manage your accounts in one central location - the Azure portal.

Add ZyLAB ONE Legal Hold from the Gallery

Note: To configure the integration of ZyLAB ONE Legal Hold into Azure AD, you need to add ZyLAB ONE Legal Hold from the gallery to your list of managed SaaS apps.

  1. Sign in to the Azure portal with an Azure AD subscription (Application Administrator account).

  2. Select the View button beneath the Manage Azure Active Directory tile.

  3. Select the Enterprise applications section.

  4. Select + New application.

  5. Search for 'zylab' and select the ZyLAB ONE Legal Hold tile.

  6. Select Sign up for ZyLAB ONE Legal Hold (bottom right corner).

  7. Select your Application Administrator account.

  8. Select Consent on behalf of your organization (recommended) and click Accept.
    With 'Consent on behalf of your organization' selected, custodians do not have to consent when logging in. It makes life easier for them.

  9. The ZyLAB ONE Legal Hold application is added and you will be signed in. However, since no roles have been set yet for ZyLAB ONE Legal Hold, you will be directed to the empty Custodian Portal.
    Close this tab and continue with Step 2 below to assign roles to ZyLAB ONE Legal Hold.

Assign Users and Roles to ZyLAB ONE Legal Hold

Azure Active Directory uses a concept called assignments to determine which users should receive access to selected apps. Decide which users and/or groups in Azure AD need access to ZyLAB ONE Legal Hold. Once decided, you can assign these users and/or groups to ZyLAB ONE Legal Hold by following the instructions here. The following roles can be assigned to users: System Administrator, Global Hold Manager, Global Hold Viewer , Matter Manager, Matter Viewer and Custodian.

Note 1: At a minimum, assign one user (with an Azure Application Administrator account) as System Administrator (to synchronize contacts) and one user as Global Hold Manager (to test sending preview emails).

Note 2: Custodians do not require a role in the Azure portal when the "User assignment required" is set to No. This is preferred. Users without a role assigned will only have access to the Custodian portal of ZyLAB ONE Legal Hold when added to a hold. If you need to restrict access to the custodian portal for specific users, then you can use the custodian role assignment.

  1. Refresh the Enterprise applications section.

  2. Search for and select the ZyLAB ONE Legal Hold application.

  3. Select the 1. Assign users and groups tile.

  4. Select + Add user.

  5. Select Users and groups.

  6. In the right pane, select a user with an Azure Application Administrator account, and click Select at the bottom.

  7. Select Select a role.

  8. In the right pane, select the System Administrator role, and click Select at the bottom.

  9. In the bottom left corner, select Assign.

  10. Repeat step 4 to 9, but now assign a user with the Global Hold Manager Role.

    Note: This user does not need to have an Azure Application Administrator account.

Synchronize Contacts

In this section you'll learn how to add contacts from the Azure Active Directory to ZyLAB ONE Legal Hold.

Prerequisite: You need a user (with an Azure Application Adminstrator role) with the ZyLAB ONE Legal Hold System Administrator role.

  1. Go to https://legalhold.one (depending on previous settings, you might have another domain, like .eu)

  2. Log in with the user (with an Azure Application Administrator role) that has the ZyLAB ONE Legal Hold System Administrator role.

    • As a System Administrator you can synchronize contacts from the Azure Directory.

  3. On the Address Book page, select the Settings icon  

  4. Select Synchronize Contacts.

    • By synchronizing contacts, you will always work with an up-to-date list of potential custodians.

    • All users from the Azure Active Directory will be added. You only need to do this once. The list with users will be automatically synchronized every day.

  5. Select the user with the ZyLAB ONE Legal Hold System Administrator role.

  6. Select Accept.

    Note: To synchronize contacts, it is not required to select 'Consent on behalf of your organization' as only Legal Hold System Administrators will ever see this option. Once contacts are synchronized, the Legal Hold system administrator never has to follow these steps again as the list with users will be automatically synchronized every day. Other users of Legal Hold are not allowed to synchronize contacts.

    Once the contacts have been synchronized, close the tab.

Note: when a user is missing after synchronization, the User Type might be empty or wrongly stated. It must be 'Member'.

For more information, please refer to User Type is Blank.

Enable Sending Emails

In this section you'll learn how to enable ZyLAB ONE Legal Hold to send email messages on your behalf.

Prerequisite: You need a user with the ZyLAB ONE Legal Hold Global Hold Manager role. This user needs to have an Office 365 license with Exchange Online element enabled. Please check this at Home > Users > "User with ZyLAB ONE Legal Hold Global Hold Manager role" > Licenses.

  1. Go to https://legalhold.one (Depending on previous settings, you might have another domain, like .eu).

  2. Log in with the user that has the ZyLAB ONE Legal Hold Global Hold Manager role.

  3. In the top right user menu, you can navigate to Profile Settings.

  4. Click in the Email Preferences section on Enable, to enable email sending.

  5. Click Accept in the Permissions Requested pop-up to allow ZyLAB ONE Legal Hold to send emails on your behalf.

  6. Now you can send a test message to check if ZyLAB ONE Legal Hold is able to sent email messages. Click on Send Test Message.

  7. Click Yes to send the message. Check your Inbox to see if the message has been sent.

  8. Alternatively, as a Global Hold Manager, test sending emails using the email message preview option.

    1. In the menu on the left, select Configuration > Message Templates.

    2. Select  .

    3. Define a Name (for example, Test Message), Type, Subject and a write a bit of content. Click Save (bottom right corner).

    4. On the Message Templates page, select Preview Message Template.

    5. Select  .

    6. Select the user with the ZyLAB ONE Legal Hold Global Hold Manager role.

    7. Select Accept.
      The preview message has been sent.

You have learned how to integrate ZyLAB ONE Legal Hold with Azure Active Directory. You have synchronized your contacts and sent a preview message.
