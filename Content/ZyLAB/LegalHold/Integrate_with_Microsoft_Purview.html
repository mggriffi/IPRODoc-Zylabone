Integrate with Microsoft Purview

Preserving data in-place means you retain the data where it is actually being used, rather than moving it to a separate archiving solution or to a separate location.

Microsoft Purview (fka Microsoft 365 Compliance Center) Preservation in Place Support

LEGAL HOLD supports preservation in-place for Microsoft Purview formerly known as the Microsoft 365 Compliance Center using Standard eDiscovery. This means that for custodians that are part of a hold, you can preserve (put on hold) their Mailbox and OneDrive in Microsoft 365.

Mailbox and OneDrive preservation is bound to custodians, meaning that LEGAL HOLD will preserve the Mailbox and OneDrive for the custodians selected in your Hold.

In addition to custodian locations, you can also include in Teams, SharePoint and Shared Mailboxes to be preserved.

Create Integration with Microsoft Purview

  1. Select Create Integration.

     

  2. Select the Integration Type and the Tenant. Click Next. 

  3. Authenticate the integration. First, copy the code. Then, go to https://microsoft.com/devicelogin

  4. Paste the code and click Next.

  5. Sign in with the account that has sufficient privileges to preserve data in-place in your Microsoft 365 subscription. You should have either Compliance Administrator (formerly known as eDiscovery Manager) or eDiscovery Administrator Role permissions. 

    For more information, please refer here (only available for Administrators): 

    https://compliance.microsoft.com/compliancecenterpermissions

  6. Click Continue.

  7. Once you have authenticated, the window can be closed and you can return to Create Integration wizard.

    Click Save to finalize the integration.

  8. Once the connection is established, a test is being executed (creation and removal of an in-place hold) to check if the integration has sufficient privileges. Select the Refresh icon (to move the status from Validating to Enabled).

  9. View the validation report with the test results by clicking the Show Validation Results button.

  10. As the authentication is limited in time, the Microsoft Purview Integrations can expire. Therefore you have a renew option for the integration.

  11. With the integration completed, you can now create Preservation Locations to determine which locations need to be preserved in the hold.

Create Integration with Microsoft 365 Preservation Explorer

  1. Select Create Integration.

     

  2. Select the Integration Type and the Tenant.  

  3. Click Next.

  4. Click Save. The Validation Process will begin.

  5. Click to show the Validation Results.

  6. With the integration completed, you can now add preservation locations.

  7. Open Locations.

  8. Click .

  9. In the Preservation Location window, enter the below information:

    • Name: Define a name for the location

    • Preservation Type: Select Microsoft Purview

    • Source: Select your source

      Note: Sources are now identified as Custodial : Specific custodians or Non-Custodial : Non specific custodians -shared groups.

    • Select your Tenant and click Save.
